Function Details

ack/1

ack(Rmcp_header::#rmcp_header{}) -> binary()

Encodes a RMCP ACK packet. An ACK is a requested response to a formerly received message. To be able to associate the ACK with the message it is generated for specific values must/will be copied from the header of the received message.

ipmi/4

ipmi(Rmcp_header::#rmcp_header{}, Properties::proplists:proplist(), Req::eipmi:request(), Data::binary()) -> binary()

Encodes a RMCP IPMI packet. This encapsulates the logic of binary packet construction with authentication and checksum calculation. Please note that currently only IPMI requests are supported.

ping/2

ping(Rmcp_header::#rmcp_header{}, Asf_ping::#asf_ping{}) -> binary()

Encodes a RMCP ASF Ping packet. A Ping is usually sent to retrieve the IPMI capabilities of the far end.
